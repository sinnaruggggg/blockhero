# UI 스튜디오 PC 편집기
PC 편집기는 `tools/visual-config-studio/` 아래에 있는 관리자용 비주얼 배치 도구입니다.  
휴대폰 실기기 프리뷰와 같은 설정 JSON을 다루되, 큰 화면에서 더 빠르게 보고 수정하도록 만든 보조 편집기입니다.

## 가장 쉬운 실행 방법
저장소 루트의 [run-ui-studio.bat](/C:/www/game/blockhero_codex/run-ui-studio.bat)을 더블클릭하면 됩니다.

이 배치 파일은 아래 작업을 자동으로 합니다.
- `tools/visual-config-studio/` 폴더를 로컬 웹 서버로 실행
- 브라우저에서 `http://localhost:4173` 열기
- 이미 같은 포트에서 실행 중이면 서버는 새로 띄우지 않고 브라우저만 열기

보조 실행 파일도 있습니다.
- [run-ui-studio.bat](/C:/www/game/blockhero_codex/tools/visual-config-studio/run-ui-studio.bat)

## 실기 화면을 보면서 PC에서 작업하기
루트의 [run-ui-studio-live.bat](/C:/www/game/blockhero_codex/run-ui-studio-live.bat)을 실행하면 아래를 한 번에 처리합니다.

- UI Studio 서버 실행 (`http://localhost:4173`)
- ADB 연결된 안드로이드 기기 해상도/안전영역을 읽어 UI Studio에 자동 반영
- `scrcpy`가 설치되어 있으면 실기 미러 창을 같이 띄움

준비물:
- USB 디버깅이 켜진 안드로이드 기기 (`adb devices`에서 `device` 상태)
- `adb` 설치 (Android Platform Tools)
- 선택: `scrcpy` 설치 (실시간 폰 화면 미러링)

참고:
- 기기 자동 감지에 실패해도 UI Studio는 기본값으로 실행됩니다.
- iPhone은 ADB 대상이 아니라 자동 감지가 되지 않으므로, 기기 프리셋 또는 커스텀 값으로 맞춰 사용합니다.

## 수동 실행 방법
```powershell
cd C:\www\game\blockhero_codex\tools\visual-config-studio
python -m http.server 4173
```

그다음 브라우저에서 `http://localhost:4173`

## 처음 켠 뒤 순서
1. 상단 로그인 영역에서 관리자 이메일과 비밀번호를 입력합니다.
2. `관리자 로그인`을 누릅니다.
3. 로그인되면 드래프트와 최신 배포본이 자동으로 조회됩니다.
4. 드래프트가 없으면 `현재 게임 기본값 초안 생성`을 눌러 서버 초안을 만듭니다.
5. 앱의 UI 스튜디오에서 드래프트를 저장하면 현재 화면의 실기기 스냅샷과 요소 좌표가 같이 서버에 올라갑니다.
6. PC 편집기는 그 스냅샷이 있으면 샘플 대신 실제 캡처 화면을 배경으로 사용합니다.
7. 요소를 수정한 뒤 `드래프트 저장` 또는 `배포`를 진행합니다.

## 이 편집기로 할 수 있는 일
- `레벨`, `무한`, `대전`, `레이드` 화면의 주요 UI 요소 위치를 조정합니다.
- 마우스로 요소를 직접 드래그해서 옮깁니다.
- 인스펙터에서 `X/Y 오프셋`, `배율`, `투명도`, `Z 인덱스`, `표시 여부`, `안전영역 기준`을 수정합니다.
- 그리드와 스냅, 확대/축소, 방향키 미세 이동으로 정교하게 맞춥니다.
- 월드/레벨/레이드 단계별 배경 이미지를 바꾸거나 제거합니다.
- 드래프트 저장, 최신 배포본 불러오기, 재배포, 이전 버전 롤백 배포를 처리합니다.
- 앱에서 올라온 실기기 스냅샷이 있으면 그 이미지를 기준으로 같은 화면을 맞춰가며 편집합니다.

## 화면별 메뉴 설명

### 서버 연결
- `Supabase 주소`: 비주얼 설정과 배경 자산이 저장된 서버 주소입니다. 기본값으로 자동 채워집니다.
- `익명 키`: 읽기 요청에 쓰는 anon key입니다. 기본값으로 자동 채워집니다.
- `관리자 이메일 / 비밀번호`: 관리자 로그인에 사용합니다.
- `직접 JWT 붙여넣기`: 로그인 대신 access token을 수동으로 넣을 때만 씁니다.
- `드래프트 불러오기`: 현재 초안 상태를 가져옵니다.
- `최신 배포 불러오기`: 실제 유저가 쓰는 최신 배포본을 가져옵니다.
- `현재 게임 기본값 초안 생성`: 서버 초안이 비어 있을 때 현재 앱 기본값으로 draft를 만듭니다.
- `배포 이력 새로고침`: 최근 배포 목록을 다시 읽습니다.
- `JSON 복사`: 현재 편집 중인 JSON을 복사합니다.

### 기기 미리보기
- `대상 화면`: 레벨, 무한, 대전, 레이드 중 편집할 화면을 고릅니다.
- `기기 프리셋`: 자주 쓰는 해상도와 안전영역 조합을 고릅니다.
- `가로 폭 / 세로 높이`: 커스텀 해상도를 직접 넣을 수 있습니다.
- `상단 안전영역 / 하단 안전영역`: 노치, 상태바, 제스처 영역을 반영합니다.
- `그리드 표시`: 보조선을 켭니다.
- `그리드 스냅`: 요소가 그리드에 맞춰 이동합니다.
- `그리드 간격`: 스냅 단위를 정합니다.
- `확대 배율`: 미리보기를 확대 또는 축소합니다.

### 배경 설정
- `적용 범위`: 월드 기본 배경, 특정 레벨 배경, 레이드 단계 배경 중 하나를 고릅니다.
- `대상 값`: 예를 들어 월드 `1`, 레벨 `12`, 레이드 `3` 같은 대상 번호입니다.
- `자산 키`: 업로드한 배경 이미지의 식별자입니다.
- `틴트 색상 / 틴트 강도`: 배경 위에 덮는 색상과 강도입니다.
- `이미지 제거`: 배경 이미지를 비우고 색상/오버레이만 남깁니다.
- `배경 적용`: 현재 입력값을 설정 JSON에 반영합니다.
- `배경 초기화`: 해당 대상의 배경 오버라이드를 제거합니다.
- `자산 업로드`: 새 배경 이미지를 서버에 올립니다.

### 레이어 목록
- 현재 화면에서 조정 가능한 요소들이 위에서 아래 순서대로 보입니다.
- 요소를 클릭하면 미리보기와 인스펙터가 해당 요소로 이동합니다.
- 체크박스로 표시 여부를 바로 바꿀 수 있습니다.

### 속성 편집
- `X/Y 오프셋`: 기준 위치에서 얼마나 이동할지 정합니다.
- `배율`: 요소 크기를 키우거나 줄입니다.
- `투명도`: 요소를 반투명하게 만들 수 있습니다.
- `Z 인덱스`: 겹쳤을 때 위로 올라올 우선순위입니다.
- `표시`: 요소 숨김/노출 여부입니다.
- `안전영역 기준`: safe area를 기준으로 위치를 계산할지 정합니다.
- `위 / 아래 / 왼쪽 / 오른쪽`: 1픽셀 단위 미세 이동 버튼입니다.

### 선택 요소 설명
- 지금 선택한 요소가 실제 게임에서 어떤 역할을 하는지 설명합니다.
- 어느 위치에 두면 좋은지, 어떤 UI와 겹치면 안 되는지도 같이 안내합니다.

### 배포 메모
- 이번 배포에서 무엇을 바꿨는지 적는 영역입니다.
- 나중에 배포 이력에서 변경 내용을 빠르게 파악할 때 씁니다.

### 원본 JSON
- 현재 편집 중인 시각 설정의 원본입니다.
- 대량 수정이나 diff 비교가 필요할 때 직접 수정할 수 있습니다.
- 잘못된 JSON을 넣으면 레이아웃이 깨질 수 있으므로, 보통은 인스펙터 조작을 우선 권장합니다.

### 배포 이력
- 최근 배포된 비주얼 설정 버전 목록입니다.
- `이 버전 열기`: 해당 버전을 편집기에서 다시 불러옵니다.
- `이 버전으로 롤백 배포`: 해당 버전을 복사해 새 배포본으로 올립니다.

## PC 편집기만으로 최종 확정하면 안 되는 이유
PC 편집기는 브라우저에서 돌아가므로, 실제 React Native 화면과 완전히 같지는 않습니다.

- 폰트 렌더링이 다를 수 있습니다.
- safe area 계산이 브라우저와 기기에서 다를 수 있습니다.
- 그림자, 배경 처리, 이미지 비율이 조금씩 다를 수 있습니다.

그래서 권장 순서는 아래와 같습니다.
1. PC 편집기로 빠르게 조정
2. 드래프트 저장
3. 휴대폰 UI 스튜디오 fullscreen preview로 최종 확인
4. 문제 없으면 배포
